第一阶段

哲学根基/古今有别

"本体"（Ontology）最初是哲学概念，源自古希腊，研究"存在"（being）的本质。在进入计算机科学之前，它在哲学中被讨论了两千多年。理解这段历史不是学术装饰——它解释了为什么今天我们用"类"和"属性"来组织知识时，背后有一种深层的思维传统。

值得注意的是，现在科学，尤其是 Palantir 公司在大模型和智能体背景下所提出的 Ontology，已经不再是哲学意义上的 Ontology，而是指一种知识组织和管理的技术手段。

第二阶段

从哲学到计算

1980-1990年代，AI 研究者发现哲学本体论可以解决知识共享问题。Gruber 1993 给出了"本体是概念化的明确规范"这一经典定义（也是后续 Neuhaus 2018 论文的核心靶子），Guarino 1998 澄清了"本体"与"概念化"的区别，Uschold & Gruninger 1996 给出了第一套工程方法论。

↳ 哲学讨论了"什么存在"，计算机把它写成"可执行的规范"。
第三阶段 · 含核心论文

概念反思

在本体被广泛使用 25 年之后，Fabian Neuhaus 在 2018 年回头追问：Gruber 那个被每个人引用的经典定义，到底经得起推敲吗？他的答案是：不。这篇论文不仅推翻了经典定义，还给出了一个全新的、可操作的本体定义——它直接影响了 LLM 时代对"本体"的理解方式。

↳ 定义已被使用 15-25 年，是时候检验它是否经得起推敲了。
What is an Ontology?
Fabian Neuhaus
2018

这篇论文的核心使命只有一个：推翻 Gruber 1993 那个被引用了 25 年的经典定义，并给出一个真正可用的替代方案。以下只讲这篇论文最重要的两个贡献：定义问题和注释问题。

核心概念一：为什么 Gruber 的定义是"智识安慰剂"？

Gruber 1993 的定义是："本体是对概念化（conceptualization）的规范（specification）"。听起来很专业，但 Neuhaus 指出——这个定义和"用光速定义英寸"是同一个逻辑错误。

这里的逻辑是这样的。中世纪英国用"三颗大麦粒首尾相接的长度"来定义一英寸。到了 17 世纪，有人觉得大麦粒不准，提议改用"光在真空中走过的距离"——但那个时代没人能测量光速。把一个模糊的概念（大麦粒/本体）用一个更难把握的概念（光速/概念化）来定义，理论上是进步的，实践上是无用的。

Gruber 定义的问题一模一样。"概念化"是什么？Neuhaus 翻遍了文献，找到了至少 5 种互相矛盾的"概念化"定义：有人说它是"领域中的对象和关系"，有人说它是"对现实片段的结构化解释"，还有人说它是"世界状态的抽象"……连最核心的术语都没有共识，用它来定义"本体"怎么可能有效？

Neuhaus 将这类定义称为"贫瘠定义"（barren definition）——因为你拿着它，仍然无法判断一个给定的东西到底是不是本体。一个好的定义必须是"肥沃"的（fertile）：接受这个定义的人，能真正用它来解释现象、指导实践。

Neuhaus 的替代方案：本体不是一个抽象的"规范"或"概念化"，而是一个具体的文档（document），由三个部分构成：(1) 词汇表——一份自然语言中的术语清单；(2) 注释——解释这些术语含义的自然语言说明；(3) 逻辑理论——描述术语之间逻辑关系的公理和定义。三者缺一不可，合在一起才能让"胜任的读者确定文档的预期解释"。这个定义的可操作性在于：你可以打开一个文件，检查里面有没有词汇、有没有注释、有没有公理——而不需要去揣测一个看不见摸不着的"概念化"。

核心概念二：为什么注释比逻辑更重要？

传统认知里，本体的核心是形式化公理（axioms）——逻辑公式决定了"什么是什么"。但 Neuhaus 用一个令人拍案叫绝的例子证明，没有注释，公理就是一堆无意义符号。

考虑下面四组 OWL 公理。从纯逻辑角度看，它们完全同构——都是两个类互不相交（disjoint）：

P disjointWith R❌ 无意义符号，根本不是一个本体
Dog disjointWith GermanShepherd❌ 逻辑有效，但内容是假的（德国牧羊犬当然是狗）
Biolgy disjointWith Chenistry⚠️ 拼错了，但领域意图清楚
Biology disjointWith Chemistry✅ 正确的本体公理

这四组的逻辑结构完全一样，但我们对它们的判断天差地别。是什么让我们能做出这些判断？不是逻辑，而是词汇。词汇选择了自然语言中的词（Dog、Biology、Chemistry），这些词自带人类的背景知识，自动建立了"预期解释"。P 和 R 没有这种背景知识，所以它连本体都不是。

这就是为什么注释是本体中最被低估的组件。注释包含了：自然语言定义、示例、参考文献、图表甚至音频。Neuhaus 区分了两种注释——"断言性注释"（assertive，如定义文本——它声称了某种语义）和"非断言性注释"（如创建者姓名、警告标签——它在元层面描述本体）。本体工程中常见的错误就是注释和公理打架：逻辑上说"Dog 和 Animal 是父子类"，注释里却写着"Dog 是一种 Mammal"。

记住这个原则：本体 = 词汇（选哪些词）+ 注释（这些词什么意思）+ 公理（这些词之间什么关系）。三者权重并不相等——注释负责建立"预期解释"，没有它，再漂亮的逻辑也只是一堆符号。

一句话总结：本体不是概念化的规范，而是一个文档——由词汇、注释和逻辑理论三部分组成。定义必须"肥沃"（能用来判断对错），注释是本体成立的前提（词汇选择 = 解释的基础）。

第四阶段

本体工程与标准化

本体工程落地的三个里程碑：(1) Berners-Lee 等 2001《The Semantic Web》——将本体定位为 Web 数据互操作的基础设施，提出"机器可理解"的数据愿景；(2) Noy & McGuinness 2001《Ontology Development 101》——最实用的手把手构建教程，至今仍是入门首选；(3) OWL（W3C Web Ontology Language, 2004）——W3C 标准化本体语言，提供 RDF/OWL 技术栈，是工业级本体的事实标准。

第五阶段

本体学习与知识图谱

手工构建本体不可扩展，自动化方法随之出现：(1) 本体学习（Ontology Learning）——从文本中自动抽取概念、关系、公理。经典框架：Wong 2012 的"本体学习蛋糕"（七层：术语抽取→同义词→概念→层级→关系→规则→公理），每层难度递增；(2) 知识图谱（Knowledge Graph）——以本体为 schema 层，承载大规模实例数据。核心文献：Hogan 等 2021《Knowledge Graphs》ACM 综述（250+ 页），覆盖 KG 全栈。本体学习是 Giglou 2023（LLMs4OL）所对比的旧范式——传统 NLP 管道被 LLM 零样本方法取代。

第六阶段 · 含核心论文

LLM 时代的本体

LLM 改变了本体两个维度：构建（Giglou 2023：零样本取代 NLP 管道）和应用（Sharma 2024：本体约束 RAG 检索，提升事实准确性）。两篇互补。

LLMs4OL: Large Language Models for Ontology Learning
Giglou et al.
2023 · ISWC

这篇论文提出了用大语言模型做本体学习的框架。传统本体学习（见 Wong 2012 的七层蛋糕）依赖 NLP 管道——分词、句法分析、模式匹配，每个环节都可能引入错误。而 LLM 可以直接从文本中零样本抽取概念和关系，省去了繁琐的管道设计。

论文在多个本体学习任务上验证了 GPT-3 等模型的有效性，包括概念分类、关系抽取和本体对齐。论文还讨论了 LLM 本体学习的局限性：幻觉（hallucination）可能导致不存在的概念关系被引入本体——这个问题的解决方案正好由下一篇论文（OG-RAG）提供。

核心要点：LLM 零样本本体学习；取代传统 NLP 管道；幻觉问题需本体约束来修正。

OG-RAG: Ontology-Grounded Retrieval-Augmented Generation For Large Language Models
Sharma et al.
2024

这篇论文将本体引入 RAG（检索增强生成）框架。传统 RAG 用向量检索返回语义相近但可能事实错误的片段；OG-RAG 用领域本体约束检索过程——只有符合本体类型约束的知识才会被检索和送入 LLM。

例如在医疗领域，如果本体规定"药物"只能"治疗""疾病"，那么检索时就不会返回"药物治疗症状"这种类型不匹配的片段，即使向量相似度很高。实验在医疗和金融领域验证了事实准确性的显著提升。这篇论文展示了本体如何让 LLM 更可靠——不是因为 LLM 变得更聪明，而是因为本体为 LLM 的知识来源加上了结构化护栏。

核心要点：本体作为 RAG 的结构化约束护栏；类型约束过滤错误检索；医疗/金融领域验证有效。

第七阶段

智能体时代的本体

LLM 从问答工具进化为自主智能体，本体的角色从被动知识骨架升级为智能体规划、推理、工具调用的认知基础设施。代表工作：KG-Agent（ACL 2025）——本体嵌入智能体迭代推理循环，将知识图谱的结构化约束注入 LLM 思维链。当前方向：本体驱动智能体（Ontology-Driven Agent），也是 Palantir 等公司在生产环境中探索的核心架构。
